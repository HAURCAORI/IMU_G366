# Epson M-G366PDG0 — Interface Specification

## Device Overview
- 6-DOF IMU: 3-axis gyroscope + 3-axis accelerometer
- Gyro scale factor: **1/66 dps/LSB**
- Accel scale factor: **1/4 mg/LSB**
- Features: attitude output (Euler/quaternion), delta angle/velocity, 3×3 rotation matrix
- Device ID register value: `0x5345`
- Software reset delay: **800 ms**

---

## UART Interface

### Baud Rates
230,400 / 460,800 / 921,600 / 1,000,000 / 1,500,000 / 2,000,000 bps
Frame format: **8N1**

### Register Write (3 bytes sent)
```
[addr | 0x80]  [data_byte]  [0x0D]
```

### Register Read (3 bytes sent → 4 bytes received)
```
Send: [addr & 0x7E]  [0x00]  [0x0D]
Recv: [echo_addr]  [data_hi]  [data_lo]  [0x0D]
```

### Burst Read (most efficient for continuous sampling)
```
Send: [0x80]  [0x00]  [0x0D]
Recv: <fields...> [checksum (optional)] [0x0D]
```
Field order: `flags → temp → gyroX/Y/Z → acclX/Y/Z → deltaAngle X/Y/Z → deltaVel X/Y/Z → quat W/X/Y/Z → attitude → GPIO → counter`

Checksum: 16-bit sum of all payload words (excluding checksum field), enabled via BURST_CTRL2.

## Initialization Sequence

1. Enter Config mode: write `CMD_CONFIG` (0x02) → `MODE_CTRL_HI` (WIN0, 0x03)
2. Poll `DIAG_STAT` (0x04) until **BIT10** set → confirms config mode
3. Read `ID` (WIN0, 0x4C) → verify = `0x5345`
4. Check `DIAG_STAT` for hardware error bits
5. Software reset: write `0x80` → `GLOB_CMD_LO` (WIN1, 0x0A), wait **800 ms**
6. Configure `SIG_CTRL` (WIN1, 0x00/01) — enable gyro/accel/temp output
7. Configure `MSC_CTRL` (WIN1, 0x02/03) — set DRDY polarity/enable
8. Configure `SMPL_CTRL_HI` (WIN1, 0x05) — set output data rate
9. Configure `FILTER_CTRL_LO` (WIN1, 0x06) — set filter, poll BIT5 until ready
10. Configure `BURST_CTRL1/2` (WIN1, 0x0C–0F) — select fields, 16/32-bit mode
11. Configure `POL_CTRL` (WIN1, 0x10/11) — axis polarity (if needed)
12. Configure `ATTI_CTRL` (WIN1, 0x14/15) — attitude mode (if needed)
13. Enter Sampling mode: write `CMD_SAMPLING` (0x01) → `MODE_CTRL_HI`

---

## Important Notes

- **Window switching**: write `0x00` or `0x01` to `WIN_CTRL` (0x7E) before accessing registers.
- Configuration registers are **only writable in Config mode**.
- Burst mode is the recommended data acquisition method (minimal UART overhead).
- Every command and response is terminated with delimiter `0x0D`.
- Self-test: write `0x04` → `MSC_CTRL_HI`, poll BIT10, check `DIAG_STAT` bits 11–15 (mask `0x7800`).
